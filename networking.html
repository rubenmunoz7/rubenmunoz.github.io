Networking Concepts

This page explains the networking concepts that apply to how this website works.

DNS (Domain Name System)

DNS is like the phonebook of the internet. When you type a website URL into your browser, DNS translates that domain name into an IP address so your computer knows where to send the request.

For example, this site is hosted on GitHub Pages. When someone visits the URL, a DNS lookup happens and resolves the domain to GitHub's servers.

You can test this yourself by running:

nslookup rubenmunoz6.github.io

or:

dig rubenmunoz6.github.io

IP Addressing

Every device on the internet has an IP address. There are two versions:

  • IPv4 - A 32-bit address written in four numbers like 192.168.1.1. Most common.
  • IPv6 - A 128-bit address written in hex like 2606:50c0::1. Created because we ran out of IPv4 addresses.

GitHub Pages supports both IPv4 and IPv6. When you run nslookup you can see both returned.

HTTP vs HTTPS

HTTP (HyperText Transfer Protocol) is the protocol used to transfer web pages. However, HTTP is not encrypted so anyone can read the traffic.

HTTPS adds TLS encryption on top of HTTP. This means:

  • Data is encrypted between your browser and the server
  • The server identity is verified with a certificate
  • You see a padlock icon in the browser

This website enforces HTTPS through GitHub Pages. GitHub automatically provides a free TLS certificate and redirects all HTTP traffic to HTTPS.

HTTP Status Codes

When your browser requests a page, the server responds with a status code:

  • 200 OK - The page loaded successfully
  • 301 Moved Permanently - Used when HTTP redirects to HTTPS
  • 404 Not Found - The page does not exist
  • 500 Internal Server Error - Something went wrong on the server

Security Headers

GitHub Pages automatically sends security headers with every response. These help protect visitors from common web attacks. You can view them in your browser DevTools under the Network tab.

  • Strict-Transport-Security - Forces HTTPS
  • X-Content-Type-Options - Prevents MIME sniffing
  • X-Frame-Options - Prevents clickjacking

TCP/IP

TCP/IP is the set of protocols that the internet runs on. When you visit this website:

  1. Your browser does a DNS lookup to find the IP address
  2. A TCP connection is established with the server (3-way handshake)
  3. An HTTPS request is sent over that connection
  4. The server responds with the HTML page

In one sentence: Our website is hosted publicly, DNS resolves the domain to an IP address, and communication occurs over HTTPS using TCP/IP.
